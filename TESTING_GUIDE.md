### 4.2 아리아 라벨 및 Semantic HTML

DevTools → Elements 탭에서 다음을 확인하세요:

#### 로그인 페이지
```html
<label for="email">이메일</label>
<input id="email" type="email" />

<label for="password">비밀번호</label>
<input id="password" type="password" />

```

- [ ] 모든 입력 필드에 `<label>` 연결 (`for` 속성)
- [ ] 버튼에 명확한 텍스트 또는 `aria-label`

#### 테이블
```html
<table>
  <thead>
    <tr>
      <th>번호</th>
      <th>클라이언트명</th>
      <th>금액</th>
    </tr>
  </thead>
</table>
```

- [ ] `<table>`, `<thead>`, `<tbody>` 정확한 구조
- [ ] `<th>` 헤더 셀 사용

#### 아이콘 버튼
```html
<!-- 나쁜 예 -->
<button>❌</button>

<!-- 좋은 예 -->
<button aria-label="삭제">
  <Trash2 size={20} />
</button>
```

- [ ] 아이콘 버튼에 `aria-label` 또는 `title` 속성
- [ ] 상태 Badge에 텍스트 + 아이콘 (예: "✓ 지불" + 초록색)


검증 항목:
- [ ] 빌드 오류 없음

### 6.2 환경변수 확인

`.env.local` 또는 배포 환경에서 다음을 확인:

```bash
# 필수 환경변수 7개
NOTION_API_KEY=secret_xxxxx          ✓
NOTION_DB_INVOICES=xxxxx             ✓
NOTION_DB_ITEMS=xxxxx                ✓
NOTION_DB_TOKENS=xxxxx               ✓
ADMIN_EMAIL=admin@example.com        ✓
ADMIN_PASSWORD=secure-password       ✓
SESSION_SECRET=64-char-random        ✓ (최소 32자)
NEXT_PUBLIC_APP_URL=http://localhost:3000  ✓
```

- [ ] 모든 필수 환경변수 설정
- [ ] SESSION_SECRET은 64자 이상 (프로덕션)
- [ ] NEXT_PUBLIC_APP_URL은 배포 도메인과 일치

## 문제 해결

### 배포 후 로그인 실패
- SESSION_SECRET 환경변수 확인
- ADMIN_EMAIL, ADMIN_PASSWORD 확인
- 서버 로그에서 HMAC 검증 에러 확인

### PDF 다운로드 실패
- 브라우저 콘솔 확인 (DevTools → Console)
- html2canvas-pro 로드 확인
- 네트워크 요청 확인 (DevTools → Network)

### 노션 API 에러
- NOTION_API_KEY 유효성 확인
- 데이터베이스 ID 확인 (대소문자 구분)
- 노션 인테그레이션 권한 확인
