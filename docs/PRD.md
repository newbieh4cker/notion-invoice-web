# 노션 연동 온라인 견적서 MVP PRD

## 🎯 핵심 정보

**목적**: 프리랜서/소규모 사업자가 노션에 작성한 견적서를 클라이언트가 별도 로그인 없이 웹에서 조회하고 PDF로 다운로드할 수 있도록 함

**타겟 사용자**: 프리랜서/소규모 사업자(관리자) + 견적서를 받는 클라이언트 (비회원)

---

## 🚶 사용자 여정

### 관리자 여정 (견적서 관리)
```
1. 로그인 페이지
   ↓ [관리자 계정으로 로그인]

2. 대시보드
   ↓ [견적서 목록 조회]

3. 견적서 목록 페이지
   ↓ [특정 견적서 클릭]

4. 견적서 상세 페이지
   ↓ [공유 링크 생성 버튼 클릭]

5. 공유 링크 생성/관리 모달
   ↓ [토큰 생성 및 링크 복사]

6. 클라이언트에게 공유 링크 전달
   ↓ [로그아웃 또는 다른 견적서 조회]
```

### 클라이언트 여정 (견적서 열람)
```
1. 공유받은 링크 클릭
   ↓ [토큰 검증]

2. 토큰 유효 여부 확인

   [유효한 토큰] → 3. 견적서 열람 페이지
                  ↓ [견적서 조회]

   [만료/무효 토큰] → 오류 페이지 (재요청 안내)

3. 견적서 열람 페이지
   ↓ [PDF 다운로드 버튼 클릭] 또는 [페이지에서 조회]

4. PDF 다운로드 또는 계속 조회
   ↓ [완료]
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 견적서 목록/상세 조회 | 노션 DB에서 견적서 데이터를 조회하여 목록 및 상세 정보 표시 | 핵심 가치 제공 - 관리자가 작성한 견적서를 웹에서 조회 | 견적서 목록 페이지, 견적서 상세 페이지 |
| **F002** | 공유 링크(토큰) 생성 및 관리 | 특정 견적서에 대한 일회성 공유 링크를 토큰 기반으로 생성하고 유효기간 설정 | 비즈니스 핵심 - 클라이언트 접근 제어의 중심 | 견적서 상세 페이지, 공유 링크 관리 페이지 |
| **F003** | 클라이언트 견적서 열람 (토큰 기반) | 공유받은 토큰으로 해당 견적서만 조회 가능, 로그인 불필요 | 사용자 핵심 니즈 - 클라이언트가 쉽게 접근 | 클라이언트 견적서 열람 페이지 |
| **F004** | PDF 다운로드 | 현재 보고 있는 견적서를 PDF로 다운로드, 한글 폰트 정상 표시 | 사용자 기본 니즈 - 오프라인 확인 및 보관 | 클라이언트 견적서 열람 페이지, 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 관리자 인증 (로그인/로그아웃) | 관리자 계정 기반 로그인 및 세션 관리 | 관리자 전용 기능 보호 - 모든 견적서 관리자만 접근 가능 | 로그인 페이지, 대시보드 |
| **F011** | 토큰 유효성 검증 | 클라이언트 접근 시 토큰 검증, 만료/무효 토큰 차단 | 보안 필수 - 공유되지 않은 견적서 접근 방지 | 클라이언트 견적서 열람 페이지, 오류 페이지 |

### 3. MVP 이후 기능 (제외)

- 관리자 견적서 편집/생성 기능 (노션 DB에서만 관리)
- 이메일 자동 발송
- 통계/분석 대시보드
- 다중 관리자 및 권한 관리
- 알림 시스템
- 고급 검색/필터링

## 📄 페이지별 상세 기능

### 1. 로그인 페이지

> **구현 기능:** `F010` | **메뉴 위치:** 미로그인 상태 → 로그인

| 항목 | 내용 |
|------|------|
| **역할** | 관리자 계정 인증 전용 페이지 |
| **진입 경로** | 앱 최초 접속 또는 미로그인 상태에서 대시보드 접근 시도 |
| **사용자 행동** | 관리자가 이메일/비밀번호를 입력하여 로그인 |
| **주요 기능** | • 이메일 입력 필드<br>• 비밀번호 입력 필드<br>• 로그인 유효성 검사 (Zod 기반)<br>• **로그인 버튼** - 서버 액션으로 인증 처리<br>• 에러 메시지 표시 (잘못된 계정 정보) |
| **다음 이동** | 성공 → 대시보드, 실패 → 에러 메시지 표시 |

---

### 2. 대시보드 (관리자)

> **구현 기능:** `F001` | **메뉴 위치:** 관리자 → 대시보드

| 항목 | 내용 |
|------|------|
| **역할** | 관리자 견적서 관리의 진입점, 전체 현황 및 최근 견적서 표시 |
| **진입 경로** | 로그인 성공 후 자동 이동 또는 대시보드 메뉴 클릭 |
| **사용자 행동** | 견적서 목록을 한눈에 확인하고 개별 견적서로 이동 |
| **주요 기능** | • 전체 견적서 수 표시<br>• 최근 견적서 목록 (테이블 형식)<br>• 견적서 상태 필터 (모두, 초안, 발송, 열람, 지불)<br>• **견적서 상세 조회** 버튼 - 견적서 상세 페이지로 이동<br>• 로그아웃 버튼 |
| **다음 이동** | 견적서 클릭 → 견적서 상세 페이지, 로그아웃 → 로그인 페이지 |

### 3. 견적서 목록 페이지 (관리자)

> **구현 기능:** `F001` | **메뉴 위치:** 관리자 → 견적서 → 목록

| 항목 | 내용 |
|------|------|
| **역할** | 전체 견적서 목록 조회 및 관리, 상세 조회로 이동 |
| **진입 경로** | 대시보드의 "견적서" 메뉴 클릭 또는 메인 내비게이션 |
| **사용자 행동** | 견적서 목록을 보고 특정 견적서를 클릭하여 상세 정보 확인 |
| **주요 기능** | • 견적서 리스트 테이블 (번호, 클라이언트명, 발행일, 금액, 상태)<br>• 상태별 필터링 (초안, 발송, 열람, 지불)<br>• 클라이언트명 검색<br>• 정렬 기능 (발행일, 금액)<br>• **상세 조회** 링크/버튼 - 해당 견적서 상세 페이지로 이동 |
| **다음 이동** | 견적서 클릭 → 견적서 상세 페이지 |

| 항목 | 내용 |
|------|------|
| **역할** | 특정 견적서 상세 정보 조회, 공유 링크 생성, PDF 미리보기 |
| **진입 경로** | 견적서 목록에서 특정 견적서 클릭 또는 대시보드에서 직접 클릭 |
| **사용자 행동** | 견적서 상세 정보 확인, 공유 링크 생성 및 복사, PDF 다운로드 |
| **주요 기능** | • 견적서 헤더 정보 (번호, 발행일, 유효기간)<br>• 클라이언트 정보 (이름, 이메일, 전화)<br>• 항목 리스트 (상품/서비스, 수량, 단가, 소계)<br>• 합계 섹션 (소계, 세금, 총액)<br>• **공유 링크 생성** 버튼 - 모달 열기<br>• **PDF 다운로드** 버튼 - 현재 견적서 PDF 생성 및 다운로드<br>• **뒤로 가기** 버튼 - 견적서 목록으로 이동 |
| **다음 이동** | 공유 링크 생성 → 공유 링크 관리 모달, PDF 다운로드 → 파일 다운로드, 뒤로 가기 → 견적서 목록 |

## 🗄️ 데이터 모델

### Invoice (견적서)
노션 DB의 견적서 테이블에서 조회하는 필드들

| 필드 | 설명 |
|------|------|
| id | 고유 식별자 (노션 Page ID) |
| invoiceNumber | 견적서 번호 (예: INV-2026-001) |
| clientName | 클라이언트명 |
| clientEmail | 클라이언트 이메일 |
| clientPhone | 클라이언트 전화번호 |
| issueDate | 발행일 |
| expiryDate | 유효기간 만료일 |
| companyName | 발행 회사명 |
| totalAmount | 합계 금액 |
| status | 상태 (draft, sent, viewed, paid) |
| items | → InvoiceItem 리스트 (노션 Relation) |
| createdAt | 생성일 |
| updatedAt | 수정일 |

### InvoiceItem (견적서 항목)
노션 DB의 항목 테이블에서 조회하는 필드들

| 필드 | 설명 |
|------|------|
| id | 고유 식별자 (노션 Page ID) |
| description | 상품/서비스명 |
| quantity | 수량 |
| unitPrice | 단가 |
| totalPrice | 소계 (quantity × unitPrice) |
| invoiceId | → Invoice.id (관련 견적서) |

### AccessToken (공유 토큰)
애플리케이션 DB (또는 노션 DB)에서 관리

| 필드 | 설명 |
|------|------|
| id | 고유 식별자 (UUID) |
| token | 공유 토큰 (난수, 고유성 보장) |
| invoiceId | → Invoice.id (해당 견적서) |
| clientEmail | 클라이언트 이메일 |
| expiresAt | 토큰 만료일 |
| createdAt | 토큰 생성일 |
| lastAccessedAt | 마지막 접근일 |
| isRevoked | 무효화 여부 (boolean) |
